Actúa como un arquitecto senior de software.

Contexto:
Estoy desarrollando una app llamada “Manifiesto”. Actualmente el modelo de datos está mal acoplado y quiero hacer una REESTRUCTURACIÓN CORRECTA de la arquitectura, priorizando claridad y escalabilidad.

OBJETIVO PRINCIPAL:
Cambiar el modelo para que el núcleo sea la MALTA (Luggage / Manifiesto individual), no el viaje ni los artículos.

Nueva jerarquía obligatoria:
Usuario
 └── Viaje
      └── Maleta (Manifiesto)
           └── Artículos

Cada maleta genera su PROPIO certificado PDF + hash.
Un viaje puede tener múltiples maletas.
Cada maleta tiene sus propias fotos (abierta / cerrada).

MODELO DE DATOS DESEADO:

Maleta (Luggage):
- id
- tripId
- brand (ej: Samsonite)
- type (ej: documentada, mano)
- color (opcional)
- nickname (opcional)
- openPhotoUrl
- closedPhotoUrl
- isSealed
- isLocked
- certificateHash
- certificatePdfUrl
- isPremium (true si excede las 2 gratuitas)
- createdAt

Artículo (ManifestItem):
- id
- tripId
- luggageId  ← referencia obligatoria
- name
- category
- brand (marca del artículo)
- quantity
- estimatedValue
- serialNumber (opcional)

REGLAS IMPORTANTES:
- Los artículos NO deben tener datos de la maleta (marca, tipo, etc).
- Todo certificado es POR MALETA, no por viaje.
- El viaje solo agrupa maletas.
- Máximo 2 maletas gratis por viaje.
- La 3ra maleta debe activar paywall (solo lógica, no pagos aún).

UI DESEADA (conceptual, no diseño):
Tabs:
- Itinerario
- Maletas
  - Lista de maletas del viaje
  - Detalle de maleta:
    - Fotos abierta/cerrada

  - Botón “Agregar maleta”
- Settings

FLUJO DE USUARIO:
